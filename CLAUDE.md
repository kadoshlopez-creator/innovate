## Project Overview

PHP CMS for **Innovate**, a technology solutions company based in Panama. No framework — plain PHP 8.1+ with Composer autoloading (PSR-4). Tailwind CSS and Alpine.js via CDN. All UI text and URL slugs are in Spanish.

## Setup

```bash
composer install
cp .env.example .env          # fill in DB credentials + set SETUP_TOKEN
# Navigate to http://innovate.test/setup.php?token=YOUR_SETUP_TOKEN → creates DB, default admin + services
# Delete setup.php after install
```

Laragon serves this from `http://innovate.test` (document root = project root). The `.htaccess` at the root routes all requests to `index.php`.

## Common Commands

```bash
composer install        # install dependencies
composer dump-autoload  # rebuild autoloader after adding new classes
```

No build step — Tailwind CSS v3, Alpine.js v3, and Font Awesome 6.5 are all loaded from CDN.

## Architecture

**Request flow:** `index.php` → `config/routes.php` registers routes → `Router::dispatch()` instantiates the matching Controller and calls the action method.

**Namespace map (PSR-4, `src/` → `App\`):**
- `App\Core\` — Router, Request, Database (PDO singleton), Session, Auth, View, CSRF, RateLimiter
- `App\Models\` — thin wrappers around PDO queries; base `Model` provides `find`, `all`, `where`, `findWhere`, `delete`, `count`
- `App\Controllers\` — public site (Home, Services, Contact)
- `App\Controllers\Admin\` — admin panel (Auth, Dashboard, Services, Contacts, Settings)

**Views:** `src/Views/`. Rendered by `View::render($view, $data, $layout)`. Layouts (`main`, `admin`) wrap `$content`. View::render automatically injects the authenticated user and flash messages into every view via `extract()`. Partials are included directly with `include`.

**Database:** MySQL via PDO. Schema in `database/schema.sql`. Tables: `users`, `services`, `contacts`, `settings`.

**Admin panel** is at `/admin` (redirects to `/admin/login` if unauthenticated). All write actions are CSRF-protected.

## Routes

Public routes use Spanish URLs: `/servicios`, `/servicios/{slug}`, `/contacto`.

Admin routes are under `/admin`: `/admin/login`, `/admin/dashboard`, `/admin/servicios` (CRUD), `/admin/contactos`, `/admin/configuracion`.

The Router matches `{param}` placeholders using `([^/]+)` regex. Route handlers are `[ControllerClass::class, 'method']` pairs. No named routes — hardcode paths in views.

## Key Conventions

- All HTML output must go through `e()` — wraps `htmlspecialchars` with `ENT_QUOTES | ENT_SUBSTITUTE`
- Every POST route calls `$this->validateCSRF()` before processing
- Admin routes call `$this->requireAdmin()` as first line; destructive actions (delete) call `$this->requireRole('admin')`
- Slugs are generated by `Service::generateSlug()` — normalizes Spanish diacritics (á→a, ñ→n, etc.)
- Flash messages: `Session::flash('success'|'error', $msg)` → displayed by layout automatically
- Settings are fetched via `Setting::get($key, $default)` — cached in-memory per request; grouped as `general`, `contact`, or `social` in the DB

## Security Patterns

### Rate Limiting
Use `App\Core\RateLimiter` to prevent brute-force attacks:
```php
$key = 'login:' . $this->request->ip();
if (RateLimiter::tooManyAttempts($key, 5, 300)) {  // 5 attempts / 5 min
    $wait = RateLimiter::availableIn($key, 300);
    Session::flash('error', "Bloqueado por {$wait}s.");
    $this->redirect('/admin/login');
}
RateLimiter::hit($key);        // record attempt
RateLimiter::clear($key);      // clear on success
```

### Role-Based Access
```php
$this->requireAdmin();         // any authenticated user
$this->requireRole('admin');   // only users with role = 'admin'
Auth::is('admin');             // boolean check, useful in views
Auth::role();                  // returns current user role string
```

### ORDER BY Safety
All `Model::all()` calls validate the `$orderBy` column against a whitelist (`$allowedOrderBy`). When adding new models, override:
```php
protected static array $allowedOrderBy = ['id', 'title', 'created_at'];
```
Invalid columns silently fall back to `id`.

### Environment Variables (new)
- `SETUP_TOKEN` — required to access `setup.php` (e.g. `?token=abc123`)
- `TRUST_PROXY=true` — enables `X-Forwarded-For` support in `Request::ip()` for proxy environments

## Helpers (src/helpers.php)

- `e($str)` — HTML-escape for output
- `url($path)` — prefixes `APP_URL`
- `asset($path)` — shortcut for `url('assets/' . $path)`
- `old($key, $default)` — retrieves previous POST input from session (for form repopulation after validation failure)
- `csrf_field()` — returns `<input type="hidden" name="_csrf" value="...">` HTML string
- `nl2p($text)` — converts double newlines to `<p>` tags, single newlines to `<br>`

## Model Patterns

All models extend the abstract `Model` base class. Static methods operate via `PDO` singleton (`Database::getInstance()`). Base methods: `all($orderBy, $dir)`, `find($id)`, `findWhere($col, $val)`, `where($col, $val)`, `count()`, `delete($id)`. Subclasses add domain-specific methods (e.g., `Contact::paginate($page, $perPage)`, `Setting::setMany($pairs)`).

Models return plain associative arrays, not objects.

## Base Controller

`Controller.php` provides: `view()`, `redirect($url)`, `back()`, `validateCSRF()`, `requireAdmin()`, `requireRole($role)`, `json($data, $status)`, `validateRequired($fields, $post)`. The `$this->request` property is a `Request` instance available in every controller action.
